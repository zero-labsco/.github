# Contributing | 贡献指南

We welcome contributions from the community! Whether you're fixing bugs, adding features, or improving documentation, your help is appreciated.

我们欢迎社区的贡献！无论你是修复 bug、添加功能还是改进文档，你的帮助都非常宝贵。

---

## Getting Started | 开始贡献

### 1. Fork the Repository | 1. Fork 仓库

Click the "Fork" button at the top of any repository to create your own copy.

点击仓库顶部的 "Fork" 按钮创建你的副本。

### 2. Clone Your Fork | 2. 克隆你的 Fork

```bash
git clone https://github.com/your-username/repository-name.git
cd repository-name
```

### 3. Create a Branch | 3. 创建分支

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Changes | 4. 进行修改

Make your changes to the codebase. Please follow the existing code style and conventions.

对代码库进行修改。请遵循现有的代码风格和约定。

### 5. Commit Changes | 5. 提交更改

```bash
git add .
git commit -m "feat: add new feature"
```

### 6. Push to Your Fork | 6. 推送到你的 Fork

```bash
git push origin feature/your-feature-name
```

### 7. Create a Pull Request | 7. 创建 Pull Request

Go to the original repository and click "New Pull Request".

前往原始仓库并点击 "New Pull Request"。

## Code Style | 代码风格

Please follow the code style guidelines for each project. Specific guidelines may vary by language:

| Language | Formatter | Style Guide |
|----------|-----------|-------------|
| Dart/Flutter | `dartfmt` | [Dart Style Guide](https://dart.dev/guides/language/effective-dart/style) |
| JavaScript/TypeScript | `prettier` + `eslint` | [Airbnb JS Style Guide](https://github.com/airbnb/javascript) |
| Java | `google-java-format` | [Google Java Style Guide](https://google.github.io/styleguide/) |
| Go | `gofmt` | [Go Code Review Comments](https://github.com/golang/go/wiki/CodeReviewComments) |
| Rust | `rustfmt` | [Rust Style Guide](https://doc.rust-lang.org/style-guide/) |
| Python | `black` + `flake8` | [PEP 8](https://peps.python.org/pep-0008/) |

请遵循每个项目的代码风格指南。具体指南因语言而异：

| 语言 | 格式化工具 | 风格指南 |
|------|-----------|---------|
| Dart/Flutter | `dartfmt` | [Dart 风格指南](https://dart.dev/guides/language/effective-dart/style) |
| JavaScript/TypeScript | `prettier` + `eslint` | [Airbnb JS 风格指南](https://github.com/airbnb/javascript) |
| Java | `google-java-format` | [Google Java 风格指南](https://google.github.io/styleguide/) |
| Go | `gofmt` | [Go Code Review Comments](https://github.com/golang/go/wiki/CodeReviewComments) |
| Rust | `rustfmt` | [Rust 风格指南](https://doc.rust-lang.org/style-guide/) |
| Python | `black` + `flake8` | [PEP 8](https://peps.python.org/pep-0008/) |

## Reporting Issues | 报告问题

If you find a bug or have a feature request, please open an issue on GitHub with:

- A clear description of the problem
- Steps to reproduce (if applicable)
- Expected behavior
- Actual behavior

如果你发现 bug 或有功能请求，请在 GitHub 上打开一个 issue，包含：

- 问题的清晰描述
- 复现步骤（如适用）
- 预期行为
- 实际行为
